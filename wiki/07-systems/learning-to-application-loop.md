# Learning to Application Loop

## Purpose

Reduce the time between learning something useful and applying it in real life, work, business, fitness, or creative output.

The goal is not to collect better notes. The goal is to convert learning into behavior, decisions, assets, experiments, and finished work.

## Core Metric

How long does it take for a useful source to become one of these?

- a decision
- a task
- a shipped output
- a changed routine
- a business experiment
- a portfolio artifact
- a Sir Leo content or offer asset
- a technical skill practiced in a real project

## Default Processing Rule

Every meaningful learning source should end with an application section:

```markdown
## Application

### Use Now

What can be applied within 24-72 hours?

### Create

What asset, page, draft, script, checklist, or system should be created?

### Practice

What skill should be rehearsed or tested?

### Decide

What decision does this clarify?

### Ignore

What sounds interesting but should not become work right now?
```

## Source to Action Pipeline

```text
source
-> extracted insight
-> application option
-> task or decision
-> execution
-> review
-> wiki update
```

## Application Filters

Before creating tasks from learning material, ask:

- Does this connect to a current priority?
- Can this be applied in the next 7 days?
- Does this create an asset or capability that compounds?
- Does this improve Sir Leo, tech career, business, fitness, or systems?
- Is this just interesting, or is it useful now?

## Task Promotion Rule

Only promote an insight into a task when it has:

- a clear outcome
- a domain
- a definition of done
- a source link
- a review requirement if public, client-facing, pricing-related, or identity-sensitive

## Fast Application Examples

| Learning Source | Fast Application |
|---|---|
| YouTube video on premium positioning | Draft one Sir Leo offer angle and add it to review. |
| Kubernetes tutorial | Build one lab and write a resume bullet from it. |
| Fitness recovery article | Add one recovery experiment to the next training week. |
| Business strategy podcast | Create one validation question and one outreach task. |
| AI agent workflow article | Update the agent protocol or create one execution ticket. |

## Anti-Pattern

Do not let learning become a warehouse of summaries. If a source is useful, it should change something. If it does not change anything, mark it as reference or archive it.
